# 🚀 AI 감정일기: 실무형 JS 아키텍처 가이드

초보자님을 위해, 이번에 리팩토링된 코드가 각각 어떤 역할을 하는지 부품별로 쪼개어 설명해 드립니다. 이 구조는 실무에서 대규모 앱을 만들 때 코드의 질을 높이는 가장 기본적인 방식 중 하나입니다.

---

## 1. 🧩 `script.js` 내부 부품(모듈) 설명
실무에서는 코드가 서로 섞여서 꼬이지 않도록 **관심사 분리(Separation of Concerns)**를 합니다.

1.  **`CONFIG` (설정 매니저)**: 
    *   API 주소(`/api/analyze`)나 애니메이션 속도 같은 '고정된 값'들만 모아둔 곳입니다. 나중에 주소가 바뀌면 여기 하나만 고치면 전체가 다 바뀝니다. (실무 용어: 전역 상수)
2.  **`AppState` (데이터 보관함)**: 
    *   사용자의 현재 테마(다크/라이트)나 과거 기록(history) 등 "현재 앱이 기억해야 할 실시간 데이터"를 담는 바구니입니다.
3.  **`ApiService` (통신 가이드)**: 
    *   오직 "서버와 대화(fetch)"하는 일만 집중해서 합니다. 화면에 무엇이 그려지는지 몰라도 오직 데이터를 정확히 전달하는 역할만 수행합니다.
4.  **`UIRenderer` (화면 요리사)**: 
    *   HTML 파일에 있는 버튼과 박스들을 찾고, 화면에 텍스트 효과를 주거나 카드를 그리는 '시각적 작업'만 담당합니다.
5.  **`App` (전체 관리자)**: 
    *   이 모든 부품을 조립하고 실행하는 **메인 컨트롤러**입니다. 사용자님이 버튼을 눌렀을 때 누가 먼저 뛰어나갈지(ApiService -> UIRenderer 순서 등)를 정합니다.

## 🔍 초보자 용어 정리

### 📚 라이브러리 함수 vs 커스텀 함수
*   **라이브러리 함수(Built-in)**: 브라우저가 미리 만들어준 도구입니다.
    *   예: `fetch()` (통신), `localStorage` (저장), `document.getElementById()` (선택), `setInterval()` (반복 실행).
*   **커스텀 함수**: 우리가 이번에 직접 이름을 짓고 만든 로직입니다.
    *   예: `App.init()`, `ApiService.analyzeDiary()`, `UIRenderer.typeText()`.

### 🌍 전역 변수 vs 지역 변수
*   **전역 변수(Global)**: 파일 전체 어디서든 쓸 수 있는 변수입니다. 리팩토링된 코드에서 `CONFIG`, `ApiService`, `App` 등이 전역 변수(객체)입니다.
*   **지역 변수(Local)**: `{ }` 안에서만 정의되고 기능을 마친 뒤 사라지는 변수입니다. 
    *   *실무 팁*: 전역 변수를 최소화하고 기능 안에서 지역 변수 위주로 쓰는 것이 "깨끗하고 충돌 없는 좋은 코드"입니다.

## 🛠️ 기능 관리 가이드 (실습 예시)

### ➖ 특정 기능을 삭제하고 싶을 때 (예: 음성 인식)
매우 쉽습니다! **`App.init`** 함수 안에서 `this.initVoiceRecognition();` 라고 적힌 줄만 지우면 끝납니다. 다른 코드들을 삭제할 필요가 없어 사이트가 고장 날 위험이 현격히 줄어듭니다.

### ➕ 새로운 기능을 추가하고 싶을 때 (예: 기록 삭제 기능)
1.  **`ApiService`**에 서버로 삭제 명령을 보내는 로직을 추가합니다.
2.  **`UIRenderer`**의 카드 생성 로직에 `X` 버튼을 넣습니다.
3.  **`App`**에서 그 버튼을 눌렀을 때 작동할 함수를 연결합니다.
